Git Tutorial

  • Інформативні команди

  • git --version
    Версія установленого Git
  • git help
    Список основних команд Git
  • git help commit
    Детальна інформація
  • pwd
    папка в якій відкрита консоль

  • Конфігурація git

  • git config --list
    Список опцій
  • git config user.name
    Детальна інформація
  • git config --global user.name "Your name"
    Ім'я користувача
  • git config --global user.email "my@mail.ua"
    E-mail користувача
  • git config core.editor vim
    настройка редактора за замовчуванням
  • git config alias.st status
    скорочена команда для git status
  • git config alias.br branch
    скорочена команда для git branch

  • Робота з файлами локально

  • git init
    старт проекту
  • rm -rf .git
    видалення репозиторію
  • git status
  • git diff
    зміни, які були внесені (до git add)
  • git diff --staged
    зміни, які були внесені (після git add) 
  • git rm filename
    видалення файлу
  • git rm -rf filename/folder
    видалення файлу або папки
  • git mv oldfilename newfilename
    перейменування файлу
  • git mv filename folder/filename
    переміщення файлу

  • Коміти

  • git add .
  • git reset HEAD filename
    відміна індексації через git add змін
  • git commit -m "first commit"
    комміт
  • git commit -am "commit"
    індексація файлів і добавлення в коміт
  • git commit --amend
    приєднати нові файли до останнього коміту ( після git add filename )
  • git checkout -- filename
    відкат файлу в коміті
  • git commit --amend -m "new message"
    перейменування ост. коміту
  • git reset --soft HEAD^
    видалення останньго коміту, зміни збережені
  • git reset --hard HEAD^
    видалення останньго коміту, зміни також видалені

  • Історія комітів

  • git log
    загальна інформація
  • git log --oneline
    загальна інформація
  • git log --oneline --graph
    загальна інформація - графік
  • git log --author="DevName"
    список комітів конкретного девелопера
  • git show commitnumber
    перегляд коміта по id
  • git checkout commitnumber
    загрузка коміту по номеру з історії
  • git checkout commitnumber -- filename
    загрузка коміту по номеру з історії (конкретний файл)
  • q
    вихід з історії комітів

  • Git stash

  • git stash
    відправка змін в stash
  • git stash apply
    загрузка змін з stash
  • git stash list
    список змін, збережених в stash
  • git stash apply 1
    загрузка по номеру з stash
  • git stash push -m "message"
    добавлення інформації про зміни
  • git stash drop 1
    видалення
  • git stash pop 1
    загрузка змін і автоматичне видалення
  • git stash clear
    очистка stash

  • Робота з віддаленим репозиторієм

  • git remote add origin https://github.com/repo.git
    коннект локального репозиторію з github
  • git remote -v
    список віддалених репозиторіїв
  • git push -u origin master
    відправка данних на github
  • git push
    відправка данних на github
  • git push origin --force
    відправка данних з переписуванням файлів
  • git fetch
    загрузка данних з github
  • git pull
    загрузка данних з github + merge
  • git pull origin namebranch
    загрузка данних з github з гілки namebranch
  • git clone https://github.com/profilike/site.git
    клонування репозиторію

  • Робота з гілками проекту

  • git branch
    список гілок
  • git branch -la
    список гілок (включно з віддаленими)
  • git branch new
    нова гілка з іменем new
  • git checkout new
    загрузка гілки з іменем new
  • git checkout -b develop
    створення і перехід на гілку develop
  • git merge new
    злиття основної master гілки з гілкою new ( при виконанні команди потрібно бути на гілці master )
  • git merge --abort
    відміна злиття
  • git diff master new
    відмінності між двома гілками master i new
  • git branch -m master newmaster
    перейменування master на newmaster
  • git merge --stash branch
    merge гілок і об'єднання усіх комітів з гілки branch в один
  • git branch -d new
    видалення гілки з іменем new
  • git branch -DR
    видалення віддаленої гілки

  • Оптимізація з Gitflow

    (https://github.com/nvie/gitflow)

  • git flow init
    ініціалізація
  • git flow feature start auth
    створення і перехід на гілку feature/auth
  • git flow feature finish auth
    merge гілки feature/auth, видалення гілки feature/auth і перехід на develop
